API TimeStamp

Proyecto Microservicios

  1. EndPoint de la API: GET [project_url]/api/timestamp/:date_string?
  2. Un string de fecha es válido si puede usarse con new Date(date_string).
    El timestamp en unix tiene que ser un entero (no un string) y muestra un número de milisegundos.
    En estas pruebas se van a utilizar strings que cumplen con la ISO-8601 (por ejemplo "2016-11-20") para asegurarnos de que el timestamp es UTC
  3. Si el string está vacío, debería ser equivalente a ejecutar new Date(), y el servicio usará la hora actual.
  4. Si el string es válido la API devolverá un JSON con la siguiente estructura:
    {"unix": <date.getTime()>, "utc" : <date.toUTCString()> }
    Por ejemplo: {"unix": 1479663089000 ,"utc": "Sun, 20 Nov 2016 17:31:29 GMT"}
  5. Si el string es érroneo la API devolverá un JSON con la siguiente estructura:
    {"error" : "Fecha Errónea" }.

Ejemplo de uso:

    https://1-gabriel-fcc-timestamp-microservice.glitch.me//api/timestamp/2015-12-25

    https://1-gabriel-fcc-timestamp-microservice.glitch.me//api/timestamp/1450137600

    Ejemplo de resultado JSON:

    {"unix":1451001600000, "utc":"Fri, 25 Dec 2015 00:00:00 GMT"}
